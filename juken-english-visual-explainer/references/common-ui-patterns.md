# Common UI Patterns for Entrance-Exam English

Choose the smallest set of interactions that makes the language or reasoning visible. These are options, not a fixed screen layout.

## Shared Principles

- Pair every highlight, toggle, filter, and step with a specific learner question.
- Preserve the original English text and avoid breaking copyability.
- Use color together with labels, underlines, brackets, or line styles.
- Show the model answer and key takeaway without requiring interaction.
- Keep complete explanations in the HTML.

## Useful Patterns

### Structure Highlighting

- Highlight subjects, verbs, objects, complements, clauses, modifiers, discourse markers, and referents.
- Let the learner toggle layers when showing all annotations at once would be noisy.
- Add a compact legend.

### Evidence Mapping

- Link each question or claim to the sentence or phrase that supports it.
- For multiple-choice tasks, let the learner switch choices and see why each succeeds or fails.
- Separate explicit evidence from reasonable inference.

### Revision Comparison

- Show original and improved wording side by side.
- Highlight additions, deletions, and replacements.
- Explain why each meaningful revision improves grammar, clarity, logic, tone, or naturalness.

### Outline and Logic

- Show claim, reason, example, contrast, cause, effect, and conclusion as connected blocks.
- For long passages or essays, provide a paragraph map.
- Use arrows only when they represent an actual relationship.

### Vocabulary and Expression

- Let the learner reveal definitions, collocations, register notes, and reusable alternatives.
- Group alternatives by nuance or difficulty, not as an unranked list.

## Anti-Patterns

- Do not turn every sentence into a dense color map.
- Do not add decorative animation.
- Do not present a corrected answer without explaining the meaningful changes.
- Do not force a graph-like UI onto a task that is clearer as a structured comparison or annotated text.
