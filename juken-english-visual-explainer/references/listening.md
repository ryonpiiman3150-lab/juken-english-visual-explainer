# Listening Module

Use this module when the user provides a transcript, audio-derived text, or a listening question with enough content to analyze.

## Core Goal

Connect what is heard to meaning, evidence, and answer selection. If audio is unavailable, state that pronunciation analysis is based on the supplied transcript or description.

## Useful UI Patterns

- Transcript segments with reveal controls.
- Evidence timeline linked to question choices.
- Highlighting for reductions, linking, stress, and key words when justified.
- Choice comparison with the phrase that supports or rejects each option.
- Short replay markers only when actual audio playback is available in the environment.

## Avoid

- Pretending to hear audio that was not provided.
- Overstating pronunciation details from text alone.
- Giving only the transcript without evidence mapping.
